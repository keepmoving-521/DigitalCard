# V0.8.0 客户留资、线索分配与站内通知说明

## 公开留资

已发布名片提供咨询表单，收集姓名、手机或邮箱、需求说明和隐私授权。访客未勾选隐私授权时，前后端均拒绝提交。提交记录自动关联名片、名片归属员工、可选推荐产品、分享来源和提交时间。关联产品必须属于当前企业、处于已发布状态，并存在于当前名片的公开推荐快照中。

## 重复识别

系统将邮箱转换为小写，将电话移除空格及分隔符。同一名片下，24 小时内相同标准化联系方式的重复提交不会创建新线索，而会增加重复次数、更新时间和最新需求，并向访客明确提示已识别到重复咨询。

## 默认分配与权限

新线索默认分配给名片归属员工。企业管理员可转交给本企业其他启用员工；销售可领取分配给自己的线索，并标记已联系或无效。

| 权限点 | 用途 |
| --- | --- |
| `lead.read` | 进入线索中心；管理员查看企业线索，销售仅查看分配给自己的线索 |
| `lead.manage` | 查看企业全部线索并转交负责人 |
| `lead.claim` | 领取和处理分配给本人的线索 |
| `notification.read` | 查看和标记本人站内通知 |

列表和详情接口执行相同的数据范围校验，无权查看的其他员工线索统一返回未找到。

## 状态流转

```text
已分配 ──销售领取──> 处理中 ──完成联系──> 已联系
   │                    │
   └────管理员转交──────┘
            任一处理中状态 ──> 无效
```

管理员转交后状态回到“待领取”，领取时间清空。

## 站内通知

新线索创建时通知名片归属账户和企业管理员；转交时通知新的负责人。通知只对目标账户可见，支持未读数量和单条已读。重复提交不会反复创建新线索通知。

## 主要接口

| 方法与路径 | 用途 |
| --- | --- |
| `POST /api/v1/public/cards/{card_id}/leads` | 公开提交咨询 |
| `GET /api/v1/tenant/leads` | 查询权限范围内的线索 |
| `GET /api/v1/tenant/leads/{id}` | 查看线索详情 |
| `POST /api/v1/tenant/leads/{id}/assign` | 管理员转交线索 |
| `POST /api/v1/tenant/leads/{id}/claim` | 销售领取本人线索 |
| `POST /api/v1/tenant/leads/{id}/status` | 标记已联系或无效 |
| `GET /api/v1/tenant/notifications` | 查询本人通知和未读数 |
| `POST /api/v1/tenant/notifications/{id}/read` | 标记通知已读 |

## 暂不包含

复杂线索评分、短信通知、企业微信通知、自动轮询分配和跨企业共享。
