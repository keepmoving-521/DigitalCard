# V1.0 隐私与安全说明

## 数据最小化与用途

公开咨询仅收集姓名、手机或邮箱、需求说明以及必要的来源信息。访客必须主动同意隐私授权，信息仅用于本次咨询、客户跟进和商机管理，不应导出或用于无关营销。

企业是其员工、线索和客户数据的管理者，应在正式上线前补充企业名称、处理目的、保存期限、联系渠道和适用地区要求。

## 访问控制

所有企业数据按令牌中的 `company_id` 隔离。销售只能访问分配给自己或由自己负责的线索和客户；负责人转移后权限即时变化。前端菜单只是体验控制，后端每次请求均执行权限及数据范围校验。

密码使用 Argon2 哈希；访问令牌短期有效；刷新令牌使用 HttpOnly Cookie 并支持轮换和撤销。日志禁止记录密码、令牌和完整留资正文。

## 公开信息

公开接口只提供已发布名片、已发布产品和公开素材。员工、企业、名片或产品停用后公开访问立即受限。私有素材没有公开下载地址。

## 安全响应与发布检查

API 返回 `nosniff`、禁止嵌入、严格来源策略和禁用摄像头/麦克风/定位的安全响应头。生产环境拒绝默认密钥及本地浏览器地址。

发布前运行：

```powershell
.\scripts\security-check.ps1
.\scripts\check.ps1
```

发现疑似泄露时应立即轮换密钥、撤销账户会话、保存审计证据、评估影响范围，并按缺陷响应机制处理。

## 数据权利与删除

企业管理员可归档客户、停用员工和下线名片。涉及依法删除、导出或更正请求时，应同时处理业务数据库、上传素材、日志及到期备份，并保留必要的处理记录。
