# 营销表单与活动

## 表单与历史快照

表单字段支持单行文本、邮箱、电话、多行文本和单选下拉，可配置必填项、隐私说明和提交成功提示。`contact` 是必需字段。每次修改表单会增加修订号；报名时保存字段、隐私说明和修订号快照，因此表单后续变化不会破坏历史报名。

## 活动状态

活动先以草稿创建，发布后仅在 `[开始时间, 结束时间)` 内开放。未开始、已结束、主动关闭、表单停用或达到人数上限都会停止报名。活动可绑定名片、产品、负责人和默认分享渠道；转线索必须绑定名片与负责人。

## 风控与隐私

- 未勾选隐私授权无法提交；
- 同一活动、同一标准化联系方式 24 小时内只保留一条报名；
- 同一活动、同一 IP 摘要每分钟最多提交 5 次；
- 隐藏蜜罐字段有内容或 User-Agent 命中机器人特征时拒绝；
- IP 和联系方式用于风控时保存 SHA-256 摘要，报名业务字段按用户明确提交内容保存；
- 只有 `marketing.export` 角色可导出含联系方式的名单，销售仅有查看权限。

## 统计和追溯

活动统计包含有效报名总量、已转线索量、无效量、报名转线索率及渠道分布。每条报名记录活动、表单修订和快照、渠道、提交时间及可选线索编号；营销提交同时进入统一经营事件模型。
