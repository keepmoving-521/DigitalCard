# DigitalCard V1.3 用户使用指南

## 企业管理员首次使用

1. 登录管理端后进入“初始化向导”；
2. 完善企业名称、Logo、联系方式和简介；
3. 建立部门并录入或导入员工；
4. 配置名片模板和员工可编辑字段；
5. 编辑“我的名片”，保存草稿并发布；
6. 复制链接或二维码，在手机浏览器验证公开内容。

向导会显示完成进度并直接链接到对应配置页。草稿不会影响线上名片，只有发布才更新公开快照。

## 产品与素材

先在素材库上传并设为公开，再在产品中心创建产品、选择封面及附件并发布。已发布产品才能被名片推荐；修改名片推荐后还需重新发布名片。

## 销售处理客户

1. 客户从公开名片提交咨询；
2. 线索默认分配给名片归属员工，销售在“客户线索”领取；
3. 核实有效后点击“转为客户”；
4. 在“客户与商机”新增跟进方式、内容及下次提醒；
5. 创建商机，填写金额、预计成交日期并推进阶段；
6. 客户时间线可查看来源、留资、跟进和商机变化。

企业管理员可转移、合并和归档客户。合并前必须查看冲突预览，确认联系人、跟进和商机迁移数量。

## 经营分析

进入“经营分析”可按日期、部门、员工、名片、产品和渠道筛选。顶部指标用于观察访问、独立访客、分享、点击、留资和转化；趋势、排行与漏斗用于判断内容和渠道效果。事件样本可追溯对应名片、产品、线索和客户。

企业管理员和内容管理员可查看企业范围并导出报表；销售与普通员工只查看自己的数据且不能导出。页面默认排除机器人和内部访问，“最后更新时间”表示当前筛选结果中最近一条事件的发生时间。

## 营销表单与活动

进入“营销活动”先创建表单，再配置活动名称、公开地址、起止时间、人数、渠道，以及可选的名片、产品和负责人。发布后通过 H5 `/campaign/{公开地址}` 接收报名。活动未开始、已结束、关闭或满员时不能提交。

报名名单保留提交时的表单版本，后续修改表单不会改变历史数据。企业管理员和内容管理员可标记无效、转为线索并导出含联系方式的 CSV；销售仅可查看，不可导出敏感报名信息。活动转换线索前必须绑定名片和负责人。

## SaaS 运营

平台管理员进入“SaaS 运营”配置套餐，查看各企业员工、名片和存储用量，并调整套餐及到期时间。用量达到 80% 会提示；超限或到期后企业仍可读取、修改和导出现有数据，但不能新增超限资源。

冻结会立即阻止企业账户访问但不删除数据。注销需输入企业编号，默认进入 7 天冷静期；期满确认后逻辑冻结，数据继续保留 30 天供导出与按策略清理。临时协助必须填写原因和期限，生成的令牌只允许指定平台人员访问指定租户概览，所有访问均有日志。

## 常见问题

- “Permission is required”：当前角色没有该操作权限，请联系企业管理员；
- “有未发布修改”：线上仍是上一次快照，需要重新发布；
- 公开页不可访问：检查企业、员工、名片和产品状态；
- 素材无法删除：先解除产品引用或下线相关产品；
- 找不到客户：客户可能已转移给其他负责人、归档或合并；
- 页面异常：记录错误提示和请求编号，再交给管理员排查。
